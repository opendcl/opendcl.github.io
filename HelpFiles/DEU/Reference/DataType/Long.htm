Long-Werte sind große Ganzzahlen im Bereich −2.147.483.648 bis 2.147.483.647, die mit 4 Bytes Speicher dargestellt werden können. Da AutoLISP alle Ganzzahlwerte als kleine Ganzzahlen verarbeitet (siehe Integer), kann es erforderlich sein, die großen Ganzzahlen als Realzahlen vorzuhalten. Das können Sie erreichen, indem Sie eine ".0" an die große Ganzzahl anhängen (z.B. wird aus 12345678 der Wert 12345678.0), so dass AutoLISP diese Werte als Realzahlen interpretiert. OpenDCL kann eine als Argument übergebene Realzahl in eine große Ganzzahl konvertieren, wenn dieser Datentyp benötigt wird.
