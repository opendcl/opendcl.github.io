OpenDCL Installation

OpenDCL Studio lässt sich sehr leicht installieren. Laden Sie die gewünschte Sprachversion von der OpenDCL Studio Download Webseite herunter und führen Sie die MSI-Datei aus. Dialoge können Sie im OpenDCL Studio auch ohne AutoCAD-Lizenz erstellen und verändern. Um die gesamte Entwicklungsarbeit auf einem einzigen Computer zu leisten und beispielsweise den Quellcode und das Verhalten der Dialoge und Steuerelemente testen zu können, ist es jedoch empfehlenswert, das AutoCAD während der Entwicklung installiert ist. Das OpenDCL Studio enthält die OpenDCL Laufzeitumgebung zum Laden und Anzeigen der Dialoge innerhalb von AutoCAD.

OpenDCL sollte vorzugsweise lokal installiert werden. Während der Installation wird die OpenDCL-Laufzeitumgebung in allen unterstützten Versionen von AutoCAD eingerichtet, die auf dem Arbeitsplatz installiert sind.

OpenDCL Laufzeitumgebung

Die OpenDCL Laufzeitumgebung wird immer automatisch im Ordner LW:\<Programme-Verzeichnis>\<Gemeinsame Dateien>\OpenDCL installiert, unabhängig davon, ob Sie das OpenDCL Studio oder die OpenDCL Laufzeitumgebung installieren. Auf diesem Wege finden Sie bei der Installation von OpenDCL Studio die gleiche Situation wie beim Endkunden vor, der später nur die OpenDCL Laufzeitumgebung installieren wird (z.B. über das Installationsprogramm Ihrer OpenDCL Applikation). Versuchen Sie nicht, die ARX-Dateien in einen anderen Ordner zu kopieren und von dort aus in AutoCAD zu laden - es wird schlichtweg nicht funktionieren. Wie Sie die OpenDCL Laufzeitumgebung im Programmablauf Ihrer Applikation laden, erfahren Sie im Abschnitt Eine Anwendung entwickeln. Der Reiz der DemandLoading-Funktionalität besteht darin, dass Sie sich als Entwickler nicht um die Prozessorarchitektur und die verwendete AutoCAD-Version kümmern müssen - OpenDCL erledigt das für Sie und lädt die geeignete Laufzeitumgebung.

Bitte beachten Sie, dass die OpenDCL Laufzeitumgebung für alle installierten AutoCAD-Versionen eingerichtet wird. Installieren Sie nach der OpenDCL-Installation eine weitere AutoCAD-Version hinzu, für die Sie die OpenDCL-Funktionalität zur Verfügung stellen wollen, starten Sie anschließend den Reparaturvorgang von OpenDCL Studio bzw. der OpenDCL Laufzeitumgebung (z.B. beim Endanwender der OpenDCL-Applikation).

OpenDCL Beispiele

OpenDCL Studio liefert viele verschiedene Beispiele mit aus, die die unterschiedlichen Dialoge und Steuerelemente einschließlich ihrer Verwendung demonstrieren. Um die Beispiele zu starten, starten Sie AutoCAD und laden Sie die Datei "Samples\@AllSamples.lsp", die im länderspezifischen Unterordner Ihrer OpenDCL-Installation gespeichert ist. Die Datei kann durch Drag&Drop vom Windows Explorer in den Zeichenbereich der aktuellen Zeichnung geladen werden, um OpenDCL in Aktion treten zu lassen. Bitte beachten Sie, dass Sie keine zusätzlichen Supportpfade ergänzen müssen, um die OpenDCL-Beispiele zu laden und anzuwenden.
